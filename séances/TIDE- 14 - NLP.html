Vector space model avec scikit-learn TF-IDF¶

En recherche d'information, TFIDF, abrviation de term frequencyinverse document frequency, est un indicateur statistique qui cherche capturer l'importance d'un terme dans un document au sein d'un corpus.

Un corpus contient N documents. Un terme apparat dans n documents du corpus. pour un document donn de longueur L, un terme apparat T fois.

TFIDF est le produit de 2 facteurs :

  • term frequency (dans un document donn) :
    • $\frac{T}{L}$
  • inverse document frequency (frquence d'apparition dans le corpus):
    • $\log(\frac{N}{n})$

Dans ce modle, chaque document est reprsent par un vecteur normalis compos du calcul TFIDF pour chaque terme qu'il contient (vector space model).

La librairie scikit-learn permet de calculer les vecteurs TFIDF pour un corpus de documents.

Cette librairie comprend diffrentes classes traitant des textes:

  • CountVectorizer: implmente la tokenisation et le dcompte des occurrences
  • TfidfTransformer: transforme une matrice de comptage en une reprsentation TF ou TFIDF
  • TfidfVectorizer: quivalent CountVectorizer suivi par TfidfTransformer

Ces classes produisent des matrices creuses, c'est--dire des matrices pleines de 0 99% ou 99.9%, puisque chaque document ne contient que 100 ou 1000 termes et que l'ensemble d'un corpus peut contenir plusieurs 100K termes.

Nota bene

  • Traiter des matrices creuses ncessite une certaine attention car il est gnrallement pas possible de les transformer en numpy arrays ou pandas dataframes, cause des limitations de mmoire.

CountVectorizer¶

CountVectorizer: implmente la tokenisation des termes et le comptage des occurrences.

Il effectue le mme calcul que Counter. On peut calculer facilement le nombre de fois que chaque terme apparat dans un document, ou dans tout le corpus.

2.2 Gensim Word2Vec (démo)¶

Gensim : topic modelling for humans

Hypothses :

  • un corpus compos de documents
  • les documents sont composs de mots

L'algorithme utilise le plongement lexical (word embeddings) (i.e. le contexte local) pour produire des vecteurs pour chacun des mots :

  • prprocessing des documents : gensim.utils.simple_preprocess()
  • modlisation des mots : gensim.models.Word2Vec()

Ensuite, il est possible d'utiliser les proprits algbriques des vecteurs pour calculer et utiliser la similarit entre des mots. L'utilisation typique de gensim est d'aider trouver des synonymes ou des antonymes pour des applications ddies de text mining.

  • most_similar(positive=[...]): trouve les entits similaires une liste d'entits (synonymes)
  • most_similar(positive=[...], negative=[...]): trouve les entits similaires une liste d'entits (synonyms) et celles qui sont le moins similaires une autre (antonymes)
  • closer_than(entity1, entity2): trouve les entits plus proches de entity1 que entity2 l'est de entity1
  • doesnt_match(): trouve l'entit qui correspond le moins au autres

Il y a beaucoup d'autres techniques du type "X2Vec" : e.g., Doc2Vec, Fact2Vec.

WordCloud (démo)¶

Installation, ouvrir un terminal dans le dossier anaconda3/condabin :

  • sur PC : .\conda install -c conda-forge wordcloud
  • sur Mac : ./conda install -c conda-forge wordcloud

Wordcloud est une librairie Python qui permet de fabriquer des nuages de mots. La taille des mots dpend de leur nombre d'occurences.
